🛡️ Redirect Guard – Testseite

Lade diese Seite nach der Installation der Erweiterung, um alle Schutzfunktionen zu testen.

Erweiterung wird geprüft…
Test 1 – window.open()

Popup-Blocking

Öffnet ein Popup zu einer fremden Domain. Sollte blockiert oder zur blocked.html umgeleitet werden.

Test 2 – location.assign() NEU

location.assign() Redirect

Simuliert eine Weiterleitung via location.assign() zu einer Scam-Domain.

Test 3 – location.replace() NEU

location.replace() Redirect

Simuliert eine Weiterleitung via location.replace() – wie es Scam-Seiten oft tun.

Test 4 – history API NEU

history.pushState Domain-Wechsel

Versucht via history.pushState die Domain zu wechseln – ein neuer Angriffsvektor.

Test 5 – Meta-Refresh

Meta-Refresh-Tag

Fügt dynamisch einen <meta http-equiv="refresh"> Tag ein, der zu einer Scam-URL weiterleiten würde.

Test 6 – Scam-Link

Bekanntes Scam-URL-Muster

Klick auf einen Link mit bekanntem Scam-Muster (z.B. popads-Domain).

🔗 Scam-Link klicken
Test 7 – Inline onclick

Inline JavaScript Redirect JS

Button mit onclick-Handler der eine Weiterleitung auslöst.

Test 8 – Versteckter Iframe

Versteckter Iframe

Fügt einen unsichtbaren Iframe mit einer verdächtigen URL ein – häufige Tracking-Methode.

Test 9 – Whitelist (Positiv-Test)

Erlaubter Redirect

Öffnet google.com – sollte nicht blockiert werden (Whitelist).

Test 10 – Redirect-Kette

Schnelle Redirect-Kette

Löst mehrere schnelle Weiterleitungen hintereinander aus – typisch für Scam-Seiten.
