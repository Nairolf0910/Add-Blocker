🛡️ Redirect Guard

Datenschutzerklärung · Stand: Juli 2025

1. Keine Datensammlung

Redirect Guard speichert, überträgt oder verkauft keinerlei persönliche Daten. Alle Verarbeitungen finden ausschließlich lokal auf deinem Gerät statt.

2. Was lokal gespeichert wird

Die Erweiterung speichert ausschließlich deine eigenen Einstellungen im lokalen Browser-Speicher (chrome.storage.local):

  • Aktivierungs-Status (an/aus)
  • Empfindlichkeitsstufe (niedrig / mittel / hoch)
  • Deine persönliche Whitelist (Domains, die du manuell erlaubt hast)
  • Blockierungs-Zähler (nur lokal, nur für dich)

Diese Daten verlassen deinen Browser niemals.

3. Zugriffsrechte

Die Erweiterung benötigt folgende Browser-Berechtigungen:

  • webNavigation – erkennt Weiterleitungen, bevor sie ausgeführt werden
  • tabs – schließt blockierte Tabs automatisch
  • storage – speichert deine Einstellungen lokal
  • declarativeNetRequest – blockiert bekannte Werbe-Netzwerke auf Netzwerk-Ebene

Keine dieser Berechtigungen wird genutzt, um Daten zu sammeln oder weiterzugeben.

4. Dritte / Externe Server

Redirect Guard kommuniziert mit keinen externen Servern. Es gibt keine Analytics, kein Tracking, keine Werbung.

5. Open Source

Der vollständige Quellcode ist öffentlich einsehbar auf GitHub. Jeder kann die Funktionsweise selbst prüfen.

Bei Fragen zur Datenschutzerklärung: Erstelle ein Issue auf GitHub.
